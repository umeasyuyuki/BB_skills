# ワークフロー仕様

## 入力

- 必須: `workflow`, `theme`
- 任意: `depth_mode` (`standard` or `deep`)
- 任意: `save_rate_pct`

`workflow` は `picks`, `myth`, `harms`, `intake`, `auto`。

## コマンドとworkflowの対応

- `/orchestrate-picks`: `picks`
- `/orchestrate-myth`: `myth`
- `/orchestrate-harms`: `harms`
- `/orchestrate-intake`: `intake`
- `/orchestrate-auto`: `auto`

## 深掘り判定

以下のどれかで `deep`。

- 保存率 `0.5%以上`
- 作用機序重要シグナル
- トレンドシグナル

## カテゴリ別必須項目

### picks

- ランク形式
- PFC
- 価格
- 1円あたりタンパク質量
- 特筆成分
- 成分の効果
- 作用機序の噛み砕き説明

### myth

- 主張原文
- 判定（支持/一部支持/不支持）
- どの条件なら成り立つか
- 実践アクション

### harms

- 何が問題か
- どの程度で問題化しやすいか
- 置換案

### intake

- 誰向けか
- 結論（何をどう摂るか）
- 結論を支える要素（2-3個）
- 実行条件または実行方法
- 注意点
- 詳しい作用メカニズム（必要時のみ補足）

### チーム通信ルール

| 方向 | タイミング | 内容 | 目的 |
|---|---|---|---|
| researcher → competitor-analyst | 調査序盤 | 主要キーワード、発見した切り口 | 競合分析の焦点を絞る |
| competitor-analyst → researcher | 盲点発見時 | 既存投稿が未カバーの論点 | エビデンス補強を依頼 |

### マージ仕様

Phase 1 完了後、オーケストレーターが以下を実行する:

1. 両エージェントの出力を受け取る
2. リサーチ結果と競合分析結果のトピック対応関係を整理する
3. 重複トピックを除去し、独自角度を明確にする
4. マージ結果を Phase 2 の入力パッケージとして構成する

## 出力制約

- Markdownで出力
- 画像カルーセルは10枚固定
- 1枚目は13-25文字（挑発）
- 2枚目以降は30-50文字
- 1枚1メッセージ
- 各枚に感情ラベル（感情設計列）を必ず付与する
- 総まとめ表を1画像必ず含める（9枚目に配置）
- 最終成果物は `tiktok-fit-notion-publisher` でNotion保存する
- 返答末尾に `保存対象サマリー` と `Notion保存結果` を含める

## 10枚カルーセル感情設計フレームワーク

| 枚目 | 役割 | 感情設計 |
|---|---|---|
| **1** | 挑発 — スクロール止める一撃 | ドキッ |
| **2** | 問題提起 — 「お前ら気づいてるか？」 | 不安 |
| **3** | 証拠① — 事実を淡々と突きつける | 「え、マジで？」 |
| **4** | 証拠② — 論文データで裏付け | 怒りの種 |
| **5** | 証拠③ — 比較で差を際立たせる | 比較で差が際立つ |
| **6** | 思想爆発① — BBの怒りを叫ぶ | 怒り着火 |
| **7** | 思想爆発② — BBの信念を叫ぶ | 共感・覚醒 |
| **8** | 正解① — じゃあどうすればいい？ | 「助かる」 |
| **9** | 正解② — まとめ表で信頼を固める | 信頼 |
| **10** | 宣言 — BBが何者か、次も見る理由 | 宣言・帰属 |

## 事実×思想の二段構え

- **事実パート（1-5枚目）**: 淡々と証拠を並べて信頼を積む。法的に問題のない客観的事実のみ
- **思想パート（6-7枚目）**: 事実を踏まえた上で、BBの怒りと信念を"俺らの意見"として叫ぶ。意見表明であり法的にも安全
- **実用パート（8-9枚目）**: 正しい選択肢と実践方法を渡す。保存価値を担保する
- **ブランドパート（10枚目）**: 宣言+フォローCTAでBBへの帰属を促す
