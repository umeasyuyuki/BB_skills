# TikTok Fit Trend Research

## 概要

このスキルは、**サプリメント・栄養学・筋トレ関連**のフィットネストレンドを国内外から収集・分析し、TikTok発信の企画の種となる高解像度レポートを作成する。

3つのサブエージェントを並列に起動して各領域を専任で深掘りし、メインエージェントが結果を統合してレポートを執筆する。

## コマンド

- `/research-trends`

---

## 実行アーキテクチャ

```
メインエージェント（このSKILL.mdを読んでいるあなた）
  │
  │  Phase 1: 3つのサブエージェントを並列起動（同一メッセージで3つのAgent tool呼び出し）
  │
  ├─ Agent A: 国内トレンド調査  → セクション1・2の素材を返す
  ├─ Agent B: エンタメ×筋トレ   → セクション3の素材を返す
  └─ Agent C: 海外トレンド調査  → セクション4の素材を返す
      │
      │  Phase 2: 3つの結果を受け取る
      │
      │  Phase 3: 5セクション構成のレポートを執筆 → 保存 → Notion publish
      ▼
```

### IMPORTANT: 実行手順

1. **Phase 1** — 以下の「サブエージェント指示書」に記載されたプロンプトを使い、Agent toolで3つのサブエージェントを**同一メッセージ内で並列に起動**すること。すべて `subagent_type: "general-purpose"` で foreground 実行する。
2. **Phase 2** — 3つの結果が返ってきたら、各エージェントの調査結果を確認する。
3. **Phase 3** — 後述の「出力仕様」に従い、メインエージェント自身が5セクション構成のMarkdownレポートを執筆する。以下のルールを厳守すること:
   - サブエージェントの結果をそのままコピーするのではなく、**背景・理由の因果関係を補強し、セクション間の整合性を取りながら書き直す**こと。
   - **セクション間の重複排除を必ず行うこと。** 同じトレンドワード（例: クレアチン、GLP-1等）がセクション1（国内トレンド）とセクション4（海外ニュース）の両方に出現する場合、以下のように切り分ける:
     - **国内セクション**: 国内での受容状況・メディア報道・消費者動向を中心に記述
     - **海外セクション**: その話題を除外し、国内セクションでは取り上げていない別のトレンドを選定する
   - つまり **セクション4（海外）はセクション1・2と異なるトピックを3つ選ぶこと**。Agent Cの調査結果にセクション1・2と重複するトピックしかない場合は、Agent Cの結果の中からまだ取り上げていないトピックを優先的に採用するか、重複トピックでも「海外固有の動き（規制変更、大型M&A、新カテゴリ創出など）」に焦点を絞って差別化すること。
4. **Phase 4** — `Research_report/trend_research_{YYYYMMDD}.md` に保存し、`notion_publish.py` でNotionへ保存する。

### Agent A: 国内トレンド調査

以下のプロンプトをそのまま Agent tool の `prompt` に渡すこと。

```
■ 対象ソース

  → 市場全体の俯瞰。富士経済・矢野経済のレポートや日経記事を拾う
```

## 出力仕様（メインエージェントがレポートを執筆する際の構成）

サブエージェント3つの結果を統合し、以下の **5セクション構成** でレポートを執筆する。

各セクションは「何がトレンドか」だけでなく、**なぜトレンドなのか（背景・理由）** を必ず具体的に論述すること。サブエージェントの出力をそのままコピーせず、メインエージェントの視点で因果関係を補強し、セクション間の整合性・重複排除を行うこと。

### セクション1: `## 1. 国内3大トレンドとその背景`

Agent A の「現トレンド（3件）」を元に執筆。各トレンドについて:

```markdown
### 1-1. 〈トレンドワード〉

**何が起きているか:**
- 具体的な現象（SNSでの投稿数、メディア露出、検索ボリュームの変化、新商品の発売動向など）を数値や事例とともに記述する。

**なぜトレンドになったか:**
- そのワードが注目される背景・構造的な理由を論理的に説明する。
- 例: 学術論文の発表 → メディア報道 → SNS拡散という流れ、消費者の課題感の変化、海外からの波及、著名人の発信など。

**主要な情報源:**
```

## エラーハンドリング

- サブエージェントの1つが有用な結果を返せなかった場合: 残りのエージェントの結果でレポートを作成し、該当セクションに「※調査未完了 — 有用な検索結果が得られませんでした」と明記する。
- Notion保存が失敗した場合: ローカルのMDファイルは保存済みなので、エラーメッセージをユーザーに報告する。

## 品質基準

- **解像度**: 「〇〇がトレンド」で終わらせず、「なぜトレンドなのか」「何がきっかけか」「どのくらいの規模か」まで掘り下げること。
- **具体性**: 数値（検索ボリューム、市場規模、成長率、再生回数、いいね数など）を可能な限り含めること。
- **論理性**: 各トレンドの背景を「原因→結果」の因果関係で説明すること。推測と事実を明確に区別すること。
- **実用性**: TikTok企画の種として使える情報であること。投稿に直結する切り口・角度を意識すること。
- **ファクトチェック**: URLは実在確認済みのもののみ記載。不確実な情報には「※未確認」と明記すること。
