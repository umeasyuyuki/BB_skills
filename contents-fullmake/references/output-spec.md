# 出力仕様

`/contents-fullmake` の最終出力フォーマット。必ずこの順で返す。

## 出力順

### Phase 1.5（タイトル選択結果・最初に出力）

1. `タイトル候補20件＋評価ランキング`（TOP10 を理由込みで提示。残り10件は折りたたみ）
2. `採用タイトル（メディア別3形態）`（カルーセル1枚目／Note タイトル／X 冒頭フック）
   - ユーザーが選んだ TOP1 を 3 形態に微調整した結果を明示
   - Notion 保存時は `sections.title_suggestions` に格納

### カルーセル（メイン出力）

3. `台本`（スタイルメタデータ付きフォーマット。各スライドにテキスト・サイズ・色・感情ラベルを付与）
4. `キャプション`（★必須・3500字以上の長文キャプション。Notion 保存時は `sections.caption` に格納）
5. `素材管理表`
6. `総まとめ表（1枚分の設計）`（Markdown 表形式。1行あたり3〜4列。列名はテーマに合わせて柔軟に）
7. `根拠リンク`

### Note記事

8. `Note記事`（3000-4000字。note-writer スキルの黄金構成テンプレートに従う）

### X投稿

9. `X投稿`（既定 long モード 1500-2500字。short モードは 280字以内。`references/x-post-style.md` に従う）

### 共通

10. `薬機法・表現チェック結果`（3メディア一括）
11. `Notion保存結果`（3ページ分のURLを出力）
12. `カルーセル画像生成結果`（Phase 5 を実行した場合のみ。出力ディレクトリと生成枚数）

## メディア別タイトル微調整方針

Phase 1.5 でユーザーが選んだ確定タイトル1本を、各メディアの制約に合わせて Phase 2 の **各エージェントが自メディア用に微調整する**（オーケストレーターは確定タイトル原案を渡すだけ。微調整は各エージェントの責任）。

| メディア | 文字数制約 | 微調整の方向性 |
|---|---:|---|
| カルーセル1枚目 | 13-25 字 | 「挑発」として最も短く強く。体言止め・数字の頭出し・感嘆を活用 |
| Note タイトル | 30-40 字 | SEO ワード を1〜2語含める。検索意図に答える具体性を入れる |
| X 冒頭フック（long） | 25-40 字 | 1行目に置く挑発。改行で間を作り、2行目で展開する設計 |
| X 冒頭フック（short） | 25-40 字 | 280字本文の先頭1行。同上 |

### 微調整時の遵守事項

- 確定タイトルの **核となるパターン**（疑問形 / 損失回避 / 数値 / 変化 / 秘密）を変えない
- 確定タイトルの **数値・固有名詞** を変えない
- 文字数だけ削る／伸ばす／語順を整える、の3操作のみ
- BB の禁止ワード（プロトコル等）を絶対に混入させない
- 詳細は `references/title-workflow.md` の「Step C: メディア別微調整」を参照

## カテゴリ配分の運用目標

- compare（比較系）: 25%
- debunk（誤認訂正系）: 25%
- ingredient（成分解説系）: 20%
- entertainment（エンタメ系）: 15%
- discovery（発見解剖系）: 15%

## 深掘り条件

以下のどれかで深掘りにする。

- 保存率が 0.5% 以上
- 作用機序を誤解すると実践を間違えやすい
- 筋トレ界隈のトレンド性が高い

深掘り判定は SKILL.md 内のオーケストレーターが自然言語で行う（ロジックの外出しはしない）。
