# X投稿生成ルール

Phase 2 の `x-post-writer` エージェントが従うルール。

## モード分岐

X投稿は **long モード（1500〜2500字）** を**既定**とする。Note誘導を最強にするため。
瞬発的なコメント誘発のみ狙う場合は **short モード（280字以内）** を使う。

## 共通の前提

- `references/writing-style.md` の全ルールを遵守（プロトコル禁止、AIっぽさ排除、言い切り）
- 日本語
- BBの口調（情熱が見える怒り、体言止め可、体験談・主観・偏見を積極的に入れる）
- 一次情報と数字（論文の数・摂取量・期間等）で殴る
- Phase 1.5 で確定したタイトルを **冒頭フックの素材** として使う（X 用に微調整可、長さ自由）

---

## long モード（1500〜2500字／Note誘導強化）

Note記事と連動する投稿はこちらを使う。**これが既定モード**。

### 構成（10ブロック）

1. **フック＋変化の予告**（200〜300字）
   - Phase 1.5 確定タイトルを冒頭1行に展開
   - 常識を覆す1行から始める
   - 書いた人自身の変化を数字で示す（例：月5000円減った、ベンチ10kg伸びた）
   - 「これを知らないとヤバい」という危機感を匂わせる

2. **自己紹介＋権威性**（150〜250字）
   - 「京大の研究者と薬剤師のチーム」「論文147本読んだ」等の根拠
   - 読者との距離を縮める個人的な書き方

3. **問題提起**（200〜300字）
   - 情弱インフルエンサーが言っていることを具体的に列挙
   - それを「全部根拠が薄い」等で否定する

4. **本論（3〜5パート）**（700〜1100字）
   - 結論の数字・数式
   - 作用機序や根拠となる研究
   - 失敗例・成功例の対比

5. **具体例・ストーリー**（150〜250字）
   - リアルな体験談、実測データ、失敗ログ
   - 読者が自分ごと化できる場面

6. **Note誘導の橋渡し**（80〜150字）
   - 「本記事はnote記事の一部」「他にもあります」等の自然な接続
   - 押し付けがましい書き方はしない

7. **Note限定の具体的追加価値**（200〜300字・★メイン導線）
   - 配布物（MDファイル、計算表、チェックリスト）
   - 深いデータ（体重別表、商品比較、食品DB）
   - 関連記事群
   - 3つ以上を箇条書きで明示する

8. **忙しい人向け代替案**（80〜150字）
   - 「記事読む時間ない人は、配布ファイルだけでOK」等
   - 読まない層にも選択肢を与える

9. **TikTok誘導（動画・ビジュアル派向け・★サブ導線）**（50〜100字）
   - 1〜2行でTikTokアカウントを紹介
   - URLではなく **@brain_bulking** という @メンション形式で書く（Xの外部リンク制約回避）
   - 例：「動画で見たい人は @brain_bulking（TikTok）に11枚のカルーセルでまとめてある」
   - 押し付けがましくせず、選択肢として提示する

10. **締め＋Note URL＋ハッシュタグ**（80〜150字）
    - BBの決め台詞（例：情弱ビジネスを、終わらせる）
    - **Note URL の挿入位置を必ず明示する**（後述の出力形式を参照）
    - ハッシュタグ2〜3個

### Note URL の扱い（★重要）

Note URL は**手動で後から挿入する**運用。以下のプレースホルダを使う：

```
【★ここにNote URLを貼る★】
```

- 投稿を生成する時点では Note 記事が未公開のため、この文字列をそのまま残す
- ユーザーが Note 公開後に、この部分を手動で実URLに置き換える
- プレースホルダは**目立つ装飾**（★や記号）で囲み、見落とさない形にする
- 投稿の**7番（Note限定価値）の後半**と**10番（締め）**の2箇所に配置すると、読者が見落とさない

### 書き方の原則

- 見出しは本文の流れを止めない程度に（`## 見出し` ではなく自然な段落分けでもOK）
- 数字・固有名詞・論文名を散りばめる
- 段落は2〜4行で区切る（スクロール読みに耐える）
- 語尾を揃えすぎない（「〜です」「〜ました」「体言止め」を混ぜる）

## short モード（280字以内／コメント誘発・瞬発力）

単発ネタ・カルーセル投稿に連動する瞬発型のみ使う。

### 構成

```
{フック（最強の1行 = Phase 1.5 確定タイトルを 25〜40 字に圧縮）}

{補足1行（数値 or 対比構造）}

{CTA or 導線}

#ハッシュタグ1 #ハッシュタグ2
```

### フックの設計原則

- カルーセルの1枚目（挑発）のテキストを凝縮する
- 「え？」と思わせる数値 or 対比構造を先頭に置く
- 句読点を最小限にし、改行で区切る

### 導線の設計

- Noteへの誘導：「Note版に配布物と一覧表あり→URL」
- カルーセルへの誘導：「カルーセルで全部見せた→プロフから」
- リンクなし：数式CTA（`A(単位) × B(単位) = ?`）で締める

---

## 出力形式

```markdown
# X投稿（mode: long / short）

{投稿本文}

---
文字数: {N}字
モード: long / short
ハッシュタグ: #xxx #yyy
導線: Note（メイン）+ TikTok（サブ）
Note URL プレースホルダ位置: ブロック7 / ブロック10
```

### Note URL 挿入位置の書き方（long モード）

出力する投稿本文内で、以下のように**2箇所にプレースホルダを置く**：

**ブロック7の末尾（Note限定価値セクション）**
```
▶︎ 体重別の摂取量早見表
▶︎ Claude Code で自動計算できるMDファイル配布
▶︎ 関連記事：「吸収率の嘘」「ホエイvsカゼイン」

【★Note公開後、ここにURLを貼る★】
```

**ブロック10（締めブロック）**
```
情弱ビジネスを、終わらせる。

Note版：【★Note公開後、ここにURLを貼る★】

#筋トレ #プロテイン
```

ユーザーは Note 記事を公開後、この2箇所の `【★Note公開後、ここにURLを貼る★】` を実URLに置換して投稿する。
