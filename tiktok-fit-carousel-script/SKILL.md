# TikTok Fit Carousel Script

## 概要

素材管理表を入力に、フィード投稿用の台本とキャプションを生成する。

## コマンド

- `/script-compare テーマ`
- `/script-ingredient テーマ`
- `/script-entertainment テーマ`
- `/script-debunk テーマ`

## 固定ルール

- 日本語で作成
- 高校生レベルで理解できる表現
- 教科書調を避ける。体言止め可
- 情弱ビジネスを終わらせるスタンス
- 思想はテーマ・ワード選択で表現する（独立した思想パートは作らない）
- 「情弱インフルエンサー」「情弱ビジネス」「論文読んだことある？」等のワードで1枚目に思想を込める
- 各スライドに感情ラベルを付与する
- 各スライドにスタイルメタデータ（テキスト・サイズ・色）を付与する

## カルーセル制約（9〜12枚可変）

### 枚数構成

| パート | 枚数 |
|---|---|
| 挑発 | 1枚 |
| 正解 | 2〜3枚 |
| 証拠 | 4〜6枚 |
| まとめ表 | 1枚 |
| CTA | 1枚 |
| **合計** | **9〜12枚** |

正解↔証拠の順序はカテゴリによって可変。テンプレートは `references/workflow-script-templates.md` を参照。

### 文字数ルール

- 固定しない。伝わる最小限で書く
- 基本3行以内。削れないなら4行まで許容
- 文字数は結果であって目的でない

### ノイズ削除ルール

**削除するワード:**

| カテゴリ | 例 |
|---|---|
| 出典の主語・人名 | 「〜の研究によると」「〜学会が」「〜のメタアナリシスでは」「Smith et al.」「〜らの研究」→ 脚注・根拠リンクに移す。本文には「12の研究が確認」「3本の論文が証明」のように**研究・論文の数量**に変換して記述する |
| 接続の前置き | 「実は」「ぶっちゃけ」「つまり」 |
| 丁寧な言い換え | 「ということがわかった」「示されている」「報告されている」 |
| 数の修飾 | 「なんと」「驚くことに」「衝撃的なことに」 |
| 体裁の接続詞 | 「そして」「また」「さらに」→ 改行で代替 |

**残すワード（核）:**

| カテゴリ | 例 |
|---|---|
| 数値と単位 | 「20〜40g」「約400%」「2〜2.5時間」 |
| 研究・論文の数量 | 「12の研究が確認」「3本の論文が証明」— 人名・著者名の代わりにこの形式で信頼性を示す。**「研究」「論文」以外の語（メタアナリシス・RCT・コホート等）は本文に使わない。脚注のみ可。** |
| 動詞の核 | 「合成は終わる」「差はゼロ」「総量が決める」 |
| 対比構造 | 「〜ではなく〜」「〜より〜が重要」 |
| BBの主語 | 「俺らは」「Brain Bulkingは」 |

## テキストスタイル定義

| 色 | 用途 | 備考 |
|---|---|---|
| 黒 | 基本テキスト | デフォルト |
| 赤 | 最重要の一語 | スライド内1〜2箇所まで。読み飛ばされたら投稿の価値が消える語のみ |
| 青 | 数値・単位 | 本文より一段階大きく |

| サイズ | 用途 |
|---|---|
| 大 | 見出し・結論・数値 |
| 中 | 本文 |
| 小 | 補足・注釈 |

## CTAルール

コメントを促す設計。保存促しではない。

### 設計公式

```
投稿テーマ関連の数値  ×  トレーニーが自慢したい数値  =  競争が生まれるCTA
```

視聴者がコメントに**数式だけ書けば済む**状態を作る。テンプレートを見れば何を書けばいいか即わかる。説明は書かない。

### 出力形式（必須）

```
{変数A}(単位) × {変数B}(単位) = ?
コメントで教えて
```

視聴者のコメントは `100 × 4 = 400` のように数字だけになり、コメント欄が自動的に比較・競争の場になる。

### トレーニー自慢数値カタログ

CTAを作るとき、このカタログから投稿テーマに最も近い1〜2カテゴリを選ぶ。

| カテゴリ | 使える変数の例 |
|---|---|
| 重量 | ベンチMAX(kg)・スクワットMAX(kg)・デッドMAX(kg) |
| 頻度 | 週のトレ回数・1回のトレ時間(分) |
| 食事量 | トレ前の米(合)・1日の卵(個)・総カロリー(kcal) |
| サプリ | 飲んでいる種類数・1回の粒数・1日の杯数 |
| プロテイン | 1日の摂取回数・1回の量(g)・体重×何g達成しているか |

### テーマ別の掛け合わせ例

| 投稿テーマ | CTAテキスト |
|---|---|
| プロテイン摂取量 | `体重(kg) × 1日摂取回数 = ?`<br>コメントで教えて |
| 炭水化物と筋合成 | `トレ前の米(合) × 週のトレ回数 = ?`<br>あなたの数値は |
| クレアチンの効果 | `クレアチン(g) × 飲み始めて何週 = ?`<br>コメントで教えて |
| デッドリフトと腰 | `デッドMAX(kg) × 体重(kg) = ?`<br>晒してほしい |
| 睡眠と筋肉回復 | `睡眠時間(h) × 週のトレ回数 = ?`<br>あなたの数値は |

### 禁止事項

- 命令形（「〜しろ」）は使わない
- 「計算してコメントしてください」等の説明は書かない（テンプレートを見れば伝わる状態にする）
- 保存を促す文言は入れない

## 台本出力フォーマット（1スライドあたり）

**まとめ表スライド以外はテキスト形式で出力する。**

```markdown
## スライド N（役割）

メイン・大・赤
テキスト行1
テキスト行2

サブ・中・黒
テキスト内容

補足・小・黒
テキスト内容

感情設計: （ラベル）
```

- `{レイヤー名}・{サイズ}・{色}` を1行目に書き、次の行からテキストを記述する
- 複数行は改行で表現する（`\n` 記法は使わない）
- レイヤー名：メイン / サブ / 補足
- サイズ：大 / 中 / 小
- 色：黒 / 赤 / 青

**まとめ表スライドのみ表形式を維持する。**

```markdown
## スライド N（まとめ表）

| 項目 | A | B |
|---|---|---|
| 比較軸1 | 値 | 値 |
| 比較軸2 | 値 | 値 |

感情設計: （ラベル）
```

## キャプション制約（★超重要・文字数絶対遵守）

- ターゲティングを明記（中級以上のトレーニー向け等）
- **文字数**: **絶対に3,500字以上**
- **キャプション構成（必須）**:
  1. 【導入】視聴者の悩みの代弁と一般的なエセ情報の否定。「なぜ俺らがこれを伝えるのか」（最低700字）
  2. 【生理学的メカニズムの本質】体内での化学反応・神経系の働きを専門用語を交えて深く解説（最低1000字）
  3. 【よくある失敗と最悪のシナリオ】間違った選択とそのデメリット（最低700字）
  4. 【実践のやり方】具体的な成分量・タイミング・食事との組み合わせ（最低800字）
  5. 【Q&A / 例外への対応】読者が抱きそうな疑問への事前回答を最低3〜4個（最低500字）
- 根拠はキャプションにも掲載
- 詳細根拠はコメント固定にも掲載
- **実在確認とファクトチェック**: 掲載する根拠リンクは架空でないこと。リンク先の内容が主張と合致しているか検証すること
- **最終チェック**: 生成したキャプションが3500字以上あるか確認。不足なら加筆

## 必須出力

**タイトル改善案は必ず最初に独立セクションとして出力する。台本の中に埋め込まない。**

1. `タイトル改善案（5つ）`（★必須・最初に出力）

```markdown
# タイトル改善案（5つ）

1. 〇〇〇〇〇
2. 〇〇〇〇〇
3. 〇〇〇〇〇
4. 〇〇〇〇〇
5. 〇〇〇〇〇
```

2. `台本`（スタイルメタデータ付きフォーマットで出力）
3. `素材管理表（投稿用整形版）`
4. `総まとめ表（1枚分の構成・表項目）`（Markdown表形式。1行あたり3〜4列。列名はテーマに合わせて柔軟に）
5. `根拠リンク（コメント欄 + キャプション用）`

出力はMarkdown。テンプレートは `references/workflow-script-templates.md` を参照。

## 文章スタイル（★全テキスト必須）

キャプション・補足テキスト・CTA本文すべてで、`../contents-fullmake/references/writing-style.md` を遵守する。

重要ルール：
- 「プロトコル」という言葉は絶対に使わない（「やり方」「実践内容」等に言い換える）
- Markdown記号（`**太字**`、`##`、`—`、全角スラッシュ `／`）を本文に残さない
- 「結論から言うと」「以下では解説します」「ステップ1：」等の説明書っぽい書き方を避ける
- 「ケースバイケース」「一概には言えませんが」で逃げない。言い切る
- 定番比喩（地図、羅針盤、土台、車の両輪、潤滑油、DNA、スパイス）を使わない
- 一次情報・主観・偏見を積極的に入れる

詳細と実務チェックリストは `../contents-fullmake/references/writing-style.md` を参照。
