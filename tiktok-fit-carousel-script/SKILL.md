# TikTok Fit Carousel Script

## 概要

素材管理表を入力に、フィード投稿用の台本とキャプションを生成する。

## コマンド

- `/script-compare テーマ`
- `/script-ingredient テーマ`
- `/script-entertainment テーマ`
- `/script-debunk テーマ`

## 固定ルール

- 日本語で作成
- 高校生レベルで理解できる表現
- 教科書調を避ける。体言止め可
- 情弱ビジネスを終わらせるスタンス
- 思想はテーマ・ワード選択で表現する（独立した思想パートは作らない）
- 「情弱インフルエンサー」「情弱ビジネス」「論文読んだことある？」等のワードで1枚目に思想を込める
- 各スライドに感情ラベルを付与する
- 各スライドにスタイルメタデータ（テキスト・サイズ・色）を付与する

## カルーセル制約（9〜12枚可変）

### 枚数構成

| パート | 枚数 |
|---|---|
| 挑発 | 1枚 |
| 正解 | 2〜3枚 |
| 証拠 | 4〜6枚 |
| まとめ表 | 1枚 |
| CTA | 1枚 |
| **合計** | **9〜12枚** |

正解↔証拠の順序はカテゴリによって可変。テンプレートは `references/workflow-script-templates.md` を参照。

### 文字数ルール

- 固定しない。伝わる最小限で書く
- 基本3行以内。削れないなら4行まで許容
- 文字数は結果であって目的でない

### ノイズ削除ルール

**削除するワード:**

| カテゴリ | 例 |
|---|---|
| 出典の主語 | 「〜の研究によると」「〜学会が」「〜のメタアナリシスでは」→ 脚注・根拠リンクに移す |
| 接続の前置き | 「実は」「ぶっちゃけ」「つまり」 |
| 丁寧な言い換え | 「ということがわかった」「示されている」「報告されている」 |
| 数の修飾 | 「なんと」「驚くことに」「衝撃的なことに」 |
| 体裁の接続詞 | 「そして」「また」「さらに」→ 改行で代替 |

**残すワード（核）:**

| カテゴリ | 例 |
|---|---|
| 数値と単位 | 「20〜40g」「約400%」「2〜2.5時間」 |
| 動詞の核 | 「合成は終わる」「差はゼロ」「総量が決める」 |
| 対比構造 | 「〜ではなく〜」「〜より〜が重要」 |
| BBの主語 | 「俺らは」「Brain Bulkingは」 |

## テキストスタイル定義

| 色 | 用途 | 備考 |
|---|---|---|
| 黒 | 基本テキスト | デフォルト |
| 赤 | 最重要の一語 | スライド内1〜2箇所まで。読み飛ばされたら投稿の価値が消える語のみ |
| 青 | 数値・単位 | 本文より一段階大きく |

| サイズ | 用途 |
|---|---|
| 大 | 見出し・結論・数値 |
| 中 | 本文 |
| 小 | 補足・注釈 |

## CTAルール

コメントを促す設計。保存促しではない。

**設計原則:** テーマから「視聴者が自分の数値・状況・経験を1つ書けば承認される問い」を毎回導出する

| 条件 | 理由 |
|---|---|
| 答えが数値・一語・一文で完結する | ハードル最小化 |
| 正解がない（自分の状況を書くだけ） | 書くリスクをゼロに |
| 書いた内容が自分のアピールになる | 承認欲求を直撃 |
| 他者の回答と比較・競争が生まれる | コメント欄が盛り上がる連鎖 |

命令形（「〜しろ」）は使わない。「教えてほしい」「聞かせてほしい」「コメントで教えて」を基本にする。

## 台本出力フォーマット（1スライドあたり）

```markdown
## スライド N（役割）

| レイヤー | テキスト | サイズ | 色 |
|---|---|---|---|
| メイン | テキスト内容 | 大/中/小 | 黒/赤/青 |
| サブ | テキスト内容 | 大/中/小 | 黒/赤/青 |
| 補足 | テキスト内容 | 小 | 黒 |

感情設計: （ラベル）
```

## キャプション制約（★超重要・文字数絶対遵守）

- ターゲティングを明記（中級以上のトレーニー向け等）
- **文字数**: **絶対に3,500字以上**
- **キャプション構成（必須）**:
  1. 【導入】視聴者の悩みの代弁と一般的なエセ情報の否定。「なぜ俺らがこれを伝えるのか」（最低700字）
  2. 【生理学的メカニズムの本質】体内での化学反応・神経系の働きを専門用語を交えて深く解説（最低1000字）
  3. 【よくある失敗と最悪のシナリオ】間違った選択とそのデメリット（最低700字）
  4. 【完全な実践プロトコル】具体的な成分量・タイミング・食事との組み合わせ（最低800字）
  5. 【Q&A / 例外への対応】読者が抱きそうな疑問への事前回答を最低3〜4個（最低500字）
- 根拠はキャプションにも掲載
- 詳細根拠はコメント固定にも掲載
- **実在確認とファクトチェック**: 掲載する根拠リンクは架空でないこと。リンク先の内容が主張と合致しているか検証すること
- **最終チェック**: 生成したキャプションが3500字以上あるか確認。不足なら加筆

## 必須出力

**タイトル改善案は必ず最初に独立セクションとして出力する。台本の中に埋め込まない。**

1. `タイトル改善案（5つ）`（★必須・最初に出力）

```markdown
# タイトル改善案（5つ）

1. 〇〇〇〇〇
2. 〇〇〇〇〇
3. 〇〇〇〇〇
4. 〇〇〇〇〇
5. 〇〇〇〇〇
```

2. `台本`（スタイルメタデータ付きフォーマットで出力）
3. `素材管理表（投稿用整形版）`
4. `総まとめ表（1枚分の構成・表項目）`（Markdown表形式。1行あたり3〜4列。列名はテーマに合わせて柔軟に）
5. `根拠リンク（コメント欄 + キャプション用）`

出力はMarkdown。テンプレートは `references/workflow-script-templates.md` を参照。
