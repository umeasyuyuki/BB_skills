# TikTok Fit Carousel Script

## 概要

素材管理表を入力に、フィード投稿用の台本とキャプションを生成する。

## コマンド

- `/script-compare テーマ`
- `/script-ingredient テーマ`
- `/script-entertainment テーマ`
- `/script-debunk テーマ`
- `/script-discovery テーマ`（異常な事例を科学で分解する10枚構成）

## 固定ルール

- 日本語で作成
- 高校生レベルで理解できる表現
- 教科書調を避ける。体言止め可
- 情弱ビジネスを終わらせるスタンス
- 思想はテーマ・ワード選択で表現する（独立した思想パートは作らない）
- 「情弱インフルエンサー」「情弱ビジネス」「論文読んだことある？」等のワードで1枚目に思想を込める
- 各スライドに感情ラベルを付与する
- 各スライドにスタイルメタデータ（テキスト・サイズ・色）を付与する
- **各スライドに感情誘発5型のいずれか1型以上を必ず組み込む**（下記「感情誘発の5型」参照）
- **感情のピーク部分には「！」を積極的に使う**（writing-style.md の「！の使い方」参照）

## 感情誘発の5型（★全スライド必須）

事実の羅列ではなく、視聴者が「すげぇ」「マジか」「知れて良かった」と感じるコピーを書く。以下5型のいずれかを各スライドに必ず1型以上組み込む。特に挑発・ツッコミ爆発・学びスライドには必須。

### 1. 裏切り型（リベール）— 常識を立てて一撃で壊す

読者が信じている常識を先に立てて、それを数字や事実で粉砕する型。

| NG（ただの事実） | OK（裏切り型） |
|---|---|
| コレステロール上限300mgは2015年撤廃 | 10年前の常識で、君はまだ卵を恐れてる |
| 全卵は卵白より吸収効率が高い | 卵白だけ食ってる君、40%損してる |
| 筋合成は体重で決まる | 気合じゃない。体重で決まる |

### 2. 桁スケール型 — 数字の大きさを体感させる

倍率・ギャップを視覚的に見せつける型。単位と対比を必ずセットで出す。

| NG | OK |
|---|---|
| コレステロール2976mg | 旧上限の10倍。それでも死亡リスクは上がらない |
| ロイシン閾値は1食3g | 3g超えた瞬間、追加したタンパクは1gも合成に使われない |
| タンパク質420g/日 | 米なら12杯分。それを毎日食ってる |

### 3. 対比型（切り返し）— 「〇〇じゃない。〇〇だ」

読者の先入観を1文で切り返す型。体言止め＋断言で決める。

| NG | OK |
|---|---|
| 16個は計算の答え | 化け物だから、じゃない。計算の答えだ |
| 量より頻度が大事 | 量の勝負じゃない。タイミングの勝負 |
| 摂れば良いわけではない | 多ければ勝ちじゃない。均等に配れ |

### 4. 自分事化型（読者を刺す）— 「君の〇〇、〜」

他人事を自分事に変える型。読者の行動・状態を直接指摘する。

| NG | OK |
|---|---|
| 1食30g以上は無駄になる | 君のプロテイン、半分は便で出てる |
| 体重÷10が目安 | 君の卵個数、体重÷10で出る！ |
| 就寝前プロテインが有効 | 君が寝てる間、筋合成は止まってる |

### 5. セルフツッコミ型（書き手が一歩引いて突っ込む）

淡々と事実を書いた直後に、書き手自身が驚き・呆れ・ツッコミを入れる型。感情のピークを作る。読者を「一緒に驚いてる仲間」にする効果がある。

| NG（淡々と終わる） | OK（ツッコミで爆発） |
|---|---|
| 管理栄養士が計算した数字 | いや、これ考えた奴頭いいだろ！ |
| 現代人の2.5倍食ってた | 江戸人、普通にアスリート飯だろ！ |
| 上限は2015年に撤廃済み | 10年前の常識で恐れてる俺ら、周回遅れすぎる！ |
| 米と大豆でアミノ酸補完 | いや、これ完全に計算されてるだろ！ |
| van Vliet 2017で確認 | 卵白派、40%損してる件。なめてた |

### セルフツッコミで使う定型フレーズ

- 「いや、〇〇だろ！」「〇〇だろ、これ！」
- 「〇〇すぎるだろ！」
- 「〇〇ってなめてた」
- 「ちょっと待ってくれ」
- 「なんで誰も〇〇してないの？」
- 「〇〇って件」「〇〇な件」
- 「マジで〇〇」「普通に〇〇」

### 型の使い分け

| スライド役割 | 推奨の型 |
|---|---|
| 挑発（1枚目） | 裏切り型 or 対比型（方向性明示＋損失＋「？？？」を叩きつける） |
| 自分事化（2枚目） | 自分事化型（「〜してませんか？ 損です」で名指し） |
| 結論予告（3枚目） | 対比型（答えの半分を出して「？？？」で残りを隠す） |
| 既知側解説 | 桁スケール型（数字で殴る） |
| **REVEAL（6枚目）** | **REVEAL型（「？？？の正体＝〇〇」）★必須** |
| 未知側解説 | 裏切り型 or 桁スケール型 |
| まとめ表 | 対比型（列の組み方で相乗効果を表現） |
| CTA | 自分事化型（視聴者を参加させる） |

## カルーセル制約

### 枚数構成（カテゴリ別）

**compare / ingredient / entertainment / debunk（10枚基準・二山構造）:**

クレアチン投稿ベンチの二山構造を採用。感情ピークを中盤REVEALとラストまとめ表の2箇所に置く。

| パート | 枚数 | 役割 |
|---|---|---|
| 挑発 | 1枚 | フック3点セット（方向性明示×謎×損失）でスワイプを起動 |
| 自分事化 | 1枚 | 「〜してませんか？ それ、損です」で読者の現状を名指し |
| 結論予告 | 1枚 | 答えの半分を出し、残り半分を「？？？」で隠す＋「答えは後ほど→」 |
| 既知側の解説 | 2枚 | 読者が想像しやすい方（A）の作用機序 |
| **REVEAL** | **1枚** | **「？？？の正体＝〇〇」— 中盤の感情ピーク** |
| 未知側の解説 | 2枚 | reveal したもう一方（B）の作用機序 |
| まとめ表 | 1枚 | 相乗効果・全体像の対照表 |
| CTA | 1枚 | 数式型 or 共感型 |
| **合計** | **10枚** | ±1枚まで可変（必須はREVEALスライド） |

**discovery（10枚固定）:**

ツカミ → 異常ポイント → 科学的解剖①② → 比較 → ツッコミ爆発①② → 学び → まとめ表 → CTA

4型とdiscoveryで構造が異なる。テンプレートは `references/workflow-script-templates.md` を参照。

### フック3点セット（★4型の1〜3枚目必須）

クレアチン投稿ベンチから抽出した、スワイプを最後まで引っ張る3点セット。1〜3枚目とタイトルに必ず組み込む。

| 要素 | 定義 | 例 |
|---|---|---|
| **方向性明示** | 「何を学べるか」の輪郭をタイトル冒頭で宣言 | `クレアチン、水で飲むのは大損！` |
| **謎フレーム（？？？）** | 答えの半分を出し、残り半分を `「？？？」` で隠す。中盤REVEALで正体を出す | `糖質×「？？？」が鍵！` |
| **損失フレーム** | 知的好奇心より強い「損したくない」心理を起動 | 「大損」「めちゃくちゃ損です」「半分も効いてない」 |

**運用ルール:**
- 1枚目（挑発）：方向性明示＋損失＋「？？？」を同時に積む
- 2枚目（自分事化）：読者の現在の行動を名指しして損失を突き付ける
- 3枚目（結論予告）：答えの半分だけ出して「答えは後ほど→」で先送り
- 6枚目（REVEAL）：「？？？」の正体を公開。メインを**大・赤**で決める

### REVEAL型スライドの書き方

感情誘発5型と並ぶ6番目の型として、中盤ピーク専用の書き方を定義する。

**定型フレーズ:**
- 「見落としがちな『？？？』の正体＝〇〇」
- 「〇〇の裏で効いてた『？？？』の正体＝〇〇」
- 「実は鍵だった『？？？』は〇〇」

**スタイル指定:**
- メイン：大・赤で「？？？の正体」を明示
- 次行：答え（成分名・軸名・注意点）を大・青 or 黒で着地
- サブ：補足1行（長文禁止。続く7〜8枚目で解説するため）

**禁止事項:**
- REVEAL枚でメカニズムまで説明しない（感情ピークを薄める）
- 複数のrevealを1枚に詰めない（1枚1reveal）

### 文字数ルール

**compare / ingredient / entertainment / debunk:**
- 固定しない。伝わる最小限で書く
- 基本3行以内。削れないなら4行まで許容
- 文字数は結果であって目的でない

**discovery（固定）:**
- 1枚目（ツカミ）: 13〜25文字
- 2〜8枚目: 30〜50文字
- 9枚目（まとめ表）・10枚目（CTA）: 通常ルール

### ノイズ削除ルール

**削除するワード:**

| カテゴリ | 例 |
|---|---|
| 出典の主語・人名 | 「〜の研究によると」「〜学会が」「〜のメタアナリシスでは」「Smith et al.」「〜らの研究」→ 脚注・根拠リンクに移す。本文には「12の研究が確認」「3本の論文が証明」のように**研究・論文の数量**に変換して記述する |
| 接続の前置き | 「実は」「ぶっちゃけ」「つまり」 |
| 丁寧な言い換え | 「ということがわかった」「示されている」「報告されている」 |
| 数の修飾 | 「なんと」「驚くことに」「衝撃的なことに」 |
| 体裁の接続詞 | 「そして」「また」「さらに」→ 改行で代替 |

**残すワード（核）:**

| カテゴリ | 例 |
|---|---|
| 数値と単位 | 「20〜40g」「約400%」「2〜2.5時間」 |
| 研究・論文の数量 | 「12の研究が確認」「3本の論文が証明」— 人名・著者名の代わりにこの形式で信頼性を示す。**「研究」「論文」以外の語（メタアナリシス・RCT・コホート等）は本文に使わない。脚注のみ可。** |
| 動詞の核 | 「合成は終わる」「差はゼロ」「総量が決める」 |
| 対比構造 | 「〜ではなく〜」「〜より〜が重要」 |
| BBの主語 | 「俺らは」「Brain Bulkingは」 |

## テキストスタイル定義

| 色 | 用途 | 備考 |
|---|---|---|
| 黒 | 基本テキスト | デフォルト |
| 赤 | 最重要の一語 | スライド内1〜2箇所まで。読み飛ばされたら投稿の価値が消える語のみ |
| 青 | 数値・単位 | 本文より一段階大きく |

| サイズ | 用途 |
|---|---|
| 大 | 見出し・結論・数値 |
| 中 | 本文 |
| 小 | 補足・注釈 |

## CTAルール

コメントを促す造り。保存促しではない。2つの型から、テーマに合う方を選ぶ。

### 型1. 数式・承認欲求型（数値勝負がハマるテーマ向け）

**使いどころ**: プロテイン量・トレ頻度・重量・摂取カロリー等、読者が自分の数値で勝負したくなるテーマ。

#### 公式

```
投稿テーマ関連の数値  ×  トレーニーが自慢したい数値  =  競争が生まれるCTA
```

視聴者がコメントに数式だけ書けば済む状態を作る。テンプレートを見れば何を書けばいいか即わかる。説明は書かない。

#### 出力形式

```
{変数A}(単位) × {変数B}(単位) = ?
コメントで教えて
```

視聴者のコメントは `100 × 4 = 400` のように数字だけになり、コメント欄が自動的に比較・競争の場になる。

#### トレーニー自慢数値カタログ

CTAを作るとき、このカタログから投稿テーマに最も近い1〜2カテゴリを選ぶ。

| カテゴリ | 使える変数の例 |
|---|---|
| 重量 | ベンチMAX(kg)・スクワットMAX(kg)・デッドMAX(kg) |
| 頻度 | 週のトレ回数・1回のトレ時間(分) |
| 食事量 | トレ前の米(合)・1日の卵(個)・総カロリー(kcal) |
| サプリ | 飲んでいる種類数・1回の粒数・1日の杯数 |
| プロテイン | 1日の摂取回数・1回の量(g)・体重×何g達成しているか |

#### テーマ別の掛け合わせ例

| 投稿テーマ | CTAテキスト |
|---|---|
| プロテイン摂取量 | `体重(kg) × 1日摂取回数 = ?`<br>コメントで教えて |
| 炭水化物と筋合成 | `トレ前の米(合) × 週のトレ回数 = ?`<br>あなたの数値は |
| クレアチンの効果 | `クレアチン(g) × 飲み始めて何週 = ?`<br>コメントで教えて |
| デッドリフトと腰 | `デッドMAX(kg) × 体重(kg) = ?`<br>晒してほしい |
| 睡眠と筋肉回復 | `睡眠時間(h) × 週のトレ回数 = ?`<br>あなたの数値は |

---

### 型2. 共感・仲間集め型（人物・嗜好・思想を晒すテーマ向け）

**使いどころ**: 格闘家・選手・推しサプリ・好きな食品等、読者が自分の選択を晒したくなるテーマ。discovery カテゴリと entertainment カテゴリで人物テーマの時に自然にハマる。

#### 公式

```
好きな{対象}コメントしてくれ！！

俺は{書き手の選択}
```

書き手が先に自分の答えを晒すことで、読者が安心して自分の答えを出せる状態を作る。「！！」を使って熱量を乗せる。

#### 出力形式

```
好きな〇〇コメントしてくれ！！
俺は〇〇
```

視聴者のコメントは `「俺は〇〇派」` のように自分の選択を晒す形になり、コメント欄が推し語り・仲間集めの場になる。

#### テーマ別の例

| 投稿テーマ | CTAテキスト |
|---|---|
| シェイドゥラエフが最強 | `好きな格闘技の選手コメントしてくれ！！`<br>`俺は萩原恭平` |
| 最強のプロテイン論 | `今飲んでるプロテインコメントしてくれ！！`<br>`俺はマイプロのインパクト` |
| 伝説のビルダー分析 | `推しのビルダーコメントしてくれ！！`<br>`俺はロニーコールマン` |
| 怪我予防の話 | `今ケアしてる部位コメントしてくれ！！`<br>`俺は左肩のローテーターカフ` |

---

### どっちを選ぶか

| テーマの性質 | 選ぶ型 |
|---|---|
| 数値で勝負できる（量・頻度・重量） | 型1. 数式型 |
| 推し・好み・選択を晒すテーマ | 型2. 共感型 |
| 迷ったら | 型1. 数式型（より普遍的に使える） |

discovery カテゴリの10枚目CTAは、テーマによって型1・型2のどちらも採用可。

### 禁止事項（両型共通）

- 命令形（「〜しろ」）は使わない。ただし「〜してくれ！！」はOK（依頼形・仲間感）
- 「計算してコメントしてください」等の説明は書かない（テンプレートを見れば伝わる状態にする）
- 保存を促す文言は入れない

## 台本出力フォーマット（1スライドあたり）

**まとめ表スライド以外はテキスト形式で出力する。テーブル形式は絶対NG。**

### ❌ 絶対NG例（テーブル形式で出力してはいけない）

```markdown
## スライド1（挑発）

| レイヤー | テキスト | サイズ | 色 |
|---|---|---|---|
| メイン | 大谷翔平が1日卵16個 | 大 | 黒 |
| サブ | 論文読んだことある？ | 中 | 赤 |
```

↑ これは**絶対に出してはいけない**。スタイルメタデータという言葉に引きずられてテーブル化しないこと。

### ✅ 正しい出力例

```markdown
## スライド1（挑発）

メイン・大・黒
大谷翔平が1日に卵を16個食べる

サブ・中・赤
化け物だから、で終わらせてる投稿、論文読んだことある？

補足・小・黒
体重÷10で君も真似できる

感情設計: ドキッ
```

### フォーマットルール

- `{レイヤー名}・{サイズ}・{色}` を1行目に書き、次の行からテキストを記述する
- レイヤーヘッダー行とテキスト行の間に改行を入れない（直接続ける）
- 次のレイヤーとの間は空行1つで区切る
- 複数行のテキストは改行で表現する（`\n` 記法は使わない、`/` で区切らない）
- レイヤー名：メイン / サブ / 補足
- サイズ：大 / 中 / 小
- 色：黒 / 赤 / 青

**まとめ表スライドのみ表形式を維持する。**

```markdown
## スライド N（まとめ表）

| 項目 | A | B |
|---|---|---|
| 比較軸1 | 値 | 値 |
| 比較軸2 | 値 | 値 |

感情設計: （ラベル）
```

## キャプション制約（★超重要・文字数絶対遵守）

- ターゲティングを明記（中級以上のトレーニー向け等）
- **文字数**: **絶対に3,500字以上**
- **キャプション構成（必須）**:
  1. 【導入】視聴者の悩みの代弁と一般的なエセ情報の否定。「なぜ俺らがこれを伝えるのか」（最低700字）
  2. 【生理学的メカニズムの本質】体内での化学反応・神経系の働きを専門用語を交えて深く解説（最低1000字）
  3. 【よくある失敗と最悪のシナリオ】間違った選択とそのデメリット（最低700字）
  4. 【実践のやり方】具体的な成分量・タイミング・食事との組み合わせ（最低800字）
  5. 【Q&A / 例外への対応】読者が抱きそうな疑問への事前回答を最低3〜4個（最低500字）
- 根拠はキャプションにも掲載
- 詳細根拠はコメント固定にも掲載
- **実在確認とファクトチェック**: 掲載する根拠リンクは架空でないこと。リンク先の内容が主張と合致しているか検証すること
- **最終チェック**: 生成したキャプションが3500字以上あるか確認。不足なら加筆

## 必須出力

**タイトル改善案は必ず最初に独立セクションとして出力する。台本の中に埋め込まない。**

### タイトル改善案（2型×2案＝計4つ・4カテゴリ共通）

4型（compare / ingredient / entertainment / debunk）共通で、以下2型を2案ずつ計4案出す。

**両型ともに「？？？」を必ず含める。** 型A は「？？？」を**答え**として問い、型B は「？？？」を**欠けてる原因**として突きつける。

#### 型A. クイズ型（ワクワクして当てたくなる）

読者が答えを予想したくなる造り。方向性明示と組み合わせて「何を学べるか」を伝える。

定型フォーマット:
- `〇〇を〜で〇〇するのは大損！〇〇×「？？？」が鍵！`
- `〇〇の本当の鍵は「？？？」— 何か分かる？`
- `〇〇を最大化する「？？？」、君は知ってる？`

例:
- `クレアチン、水で飲むのは大損！糖質×「？？？」が鍵！`
- `プロテイン吸収を2倍にする「？？？」、答えられる？`

#### 型B. 不安煽り＋言い切り型（「？？？」を欠けてる原因として突きつける）

読者の現在の行動を名指しして「損している」と断言する。そこに「？？？」を**欠けてる原因**として埋め込む。断定で刺す口調は維持しつつ、読者に「自分に足りない何か」を意識させる。

定型フォーマット:
- `〇〇してる君、「？？？」足してないから〇〇%損してる`
- `〇〇だけでやってるやつ、「？？？」知らないと一生変わらない`
- `〇〇で挫折してる原因は「？？？」。断言する`

例:
- `水でクレアチン飲んでる君、「？？？」足してないから半分も効いてない`
- `ホエイだけ飲んでる奴、「？？？」知らないから夜の8時間は筋合成ゼロ`

```markdown
3. 〇〇〇〇〇
4. 〇〇〇〇〇
```

## 文章スタイル（★全テキスト必須）

キャプション・補足テキスト・CTA本文すべてで、`../contents-fullmake/references/writing-style.md` を遵守する。

重要ルール：
- 「プロトコル」という言葉は絶対に使わない（「やり方」「実践内容」等に言い換える）
- Markdown記号（`**太字**`、`##`、`—`、全角スラッシュ `／`）を本文に残さない
- 「結論から言うと」「以下では解説します」「ステップ1：」等の説明書っぽい書き方を避ける
- 「ケースバイケース」「一概には言えませんが」で逃げない。言い切る
- 定番比喩（地図、羅針盤、土台、車の両輪、潤滑油、DNA、スパイス）を使わない
- 一次情報・主観・偏見を積極的に入れる

詳細と実務チェックリストは `../contents-fullmake/references/writing-style.md` を参照。
