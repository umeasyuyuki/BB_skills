# 投稿テンプレート

## 共通制約（4型＝compare / ingredient / entertainment / debunk）

- **10枚基準の二山構造**（挑発1 + 自分事化1 + 結論予告1 + 既知解説2 + REVEAL1 + 未知解説2 + まとめ表1 + CTA1）
- ±1枚まで可変。ただしREVEALスライドは省略不可
- 文字数固定なし。伝わる最小限。基本3行以内（削れないなら4行まで）
- 1スライド1メッセージ
- 各スライドに感情ラベルを付与
- 各スライドにスタイルメタデータ（テキスト・サイズ・色）を付与 — **必ずテキスト形式で出力。テーブル形式NG**
- 各スライドに感情誘発5型（裏切り/桁スケール/対比/自分事化/セルフツッコミ）のいずれか1型以上を必ず組み込む
- まとめ表は必ずMarkdown表形式 `| 列1 | 列2 |` で出力。1行あたり3〜4列
- 列名はテーマに合わせて柔軟に設定する（固定しない）

## フック3点セット（★1〜3枚目とタイトル必須）

| 要素 | 運用 |
|---|---|
| 方向性明示 | タイトル冒頭と1枚目で「何を学べるか」の輪郭を宣言 |
| 謎フレーム | 答えの半分を `「？？？」` で隠し、6枚目REVEALで正体を公開 |
| 損失フレーム | 「大損」「損してます」「半分も効いてない」で行動変容を迫る |

詳細は `SKILL.md` の「フック3点セット」「REVEAL型スライドの書き方」参照。

## スライド出力フォーマット（★必ずテキスト形式）

### ❌ NG（絶対やってはいけない）

```markdown
| レイヤー | テキスト | サイズ | 色 |
|---|---|---|---|
| メイン | ... | 大 | 黒 |
```

### ✅ 正しい形式

```markdown
## スライド1（挑発）

メイン・大・黒
大谷翔平が1日に卵を16個食べる

サブ・中・赤
化け物だから、で終わらせてる投稿、論文読んだことある？

感情設計: ドキッ
```

## スタイル定義（全カテゴリ共通）

| 色 | 用途 |
|---|---|
| 黒 | 基本テキスト |
| 赤 | 最重要の一語（スライド内1〜2箇所まで） |
| 青 | 数値・単位（本文より一段階大きく） |

## CTAの造り方（全カテゴリ共通）

テーマから「視聴者が自分の数値・状況・選択を1つ書けば承認される問い」を毎回導く。
命令形（「〜しろ」）は使わない。「〜してくれ！！」の依頼形・仲間感はOK。

CTAは2型から選ぶ（詳細は SKILL.md「CTAルール」参照）:
- 型1. 数式・承認欲求型 `A(単位) × B(単位) = ?` — 数値勝負テーマ向け
- 型2. 共感・仲間集め型 `好きな〇〇コメントしてくれ！！／俺は〇〇` — 人物・推し・選択テーマ向け

## compare（比較系）`〇〇 vs 〇〇`

**コンセプト転換:** 「A vs B の勝敗」ではなく「A vs B で悩むこと自体が罠、本当の鍵は『？？？』」に転換する。

**10枚構成:**

```
スライド 1（挑発）       ← `A vs B で迷ってるのは大損！本当の鍵は「？？？」`
スライド 2（自分事化）   ← `Aを選んでる君も、Bを選んでる君も…それ、損してます`
スライド 3（結論予告）   ← `結論！ 勝敗じゃなく「？？？」で決めろ！ 答えは後ほど→`
スライド 4〜5（既知解説）← AとBの表面的な差（現状の証拠パートを流用）
スライド 6（REVEAL）★  ← `見落とされてる「？？？」の正体＝〇〇（新軸：吸収率／価格/g／タイミング等）`
スライド 7〜8（未知解説）← 新軸で再評価した作用機序・データ
スライド 9（まとめ表）   ← 新軸を足した3列比較表
スライド 10（CTA）
```

**感情設計:**
- 挑発: ドキッ・「自分のこと？」
- 自分事化: 刺さる
- 結論予告: 続きが気になる
- 既知解説: データで納得
- REVEAL: 「そういうことか！」★ピーク
- 未知解説: 「マジか！」
- まとめ表: 信頼・保存したい
- CTA: 参加欲求

**CTA例:**
「どっち使ってる？量も含めて教えてほしい（例: A製品 5g）」
「今のプロテイン、1食あたりいくら？コメントで教えて」

**まとめ表例:**

| 比較項目 | A | B | 新軸（？？？） |
|---|---|---|---|
| ... | ... | ... | ... |

## ingredient（成分解説系）`牛乳 / アシュワガンダ等`

**コンセプト転換（★クレアチン投稿の直系）:** 「〇〇を単体で摂るのは大損、鍵は〇〇×『？？？』」

**10枚構成:**

```
スライド 1（挑発）       ← `〇〇を〜で摂ってるのは大損！〇〇×「？？？」が鍵`
スライド 2（自分事化）   ← `〇〇だけ飲んでませんか？…それ、効果半減です`
スライド 3（結論予告）   ← `結論！「〇〇」と「？？？」を一緒に摂れ！ 答えは後ほど→`
スライド 4〜5（既知解説）← 成分Aの作用機序（インスリン・トランスポーター等）
スライド 6（REVEAL）★  ← `見落としがちな「？？？」の正体＝〇〇`
スライド 7〜8（未知解説）← 成分Bの作用機序（ポンプ・呼び水等）
スライド 9（まとめ表）   ← 相乗効果の整理
スライド 10（CTA）
```

**感情設計:**
- 挑発: ドキッ・損失への警戒
- 自分事化: 「自分のこと？」
- 結論予告: 続きが気になる
- 既知解説: 「へえ、そういう仕組みか」
- REVEAL: 「そういうことか！」★ピーク
- 未知解説: 「マジか！」「え、そこまで効くの？」
- まとめ表: 信頼・実践したい
- CTA: 参加欲求

**CTA例:**
「一日の〇〇摂取量、コメントで教えてほしい（例: 5g / 1錠 / 飲んでない）」
「摂り始めてから何か変化あった？コメントで聞かせてほしい」

**まとめ表例:**

| 成分 | 用量 | タイミング | 相乗効果 |
|---|---|---|---|
| A | ... | ... | ... |
| ？？？ | ... | ... | ... |

## entertainment（エンタメ系）`ラランド西田 / 横川くん等`

**コンセプト転換:** 「固有名詞の逸話」ではなく「固有名詞の表の行動の裏にある『？？？』を真似しないと意味ない」に転換。

**10枚構成:**

```
スライド 1（挑発）       ← `〇〇（人物）の〇〇、表面だけ真似すると大損！鍵は「？？？」`
スライド 2（自分事化）   ← `卵16個食ってるだけの君…それ、半分も真似できてません`
スライド 3（結論予告）   ← `結論！ 〇〇がやってたのは「〇〇」と「？？？」！ 答えは後ほど→`
スライド 4〜5（既知解説）← 表の逸話の核心（固有名詞の有名なエピソード）
スライド 6（REVEAL）★  ← `実は裏でやってた「？？？」の正体＝〇〇`
スライド 7〜8（未知解説）← 科学的解剖
スライド 9（まとめ表）   ← 表の逸話 × 裏のポイント × 科学
スライド 10（CTA）
```

**口調:** 他カテゴリより柔らかくてよい。BBトーンは維持しつつ「面白い発見をした」ベース。キレ不要。

**感情設計:**
- 挑発: 「なんか面白そう」＋損失への警戒
- 自分事化: 刺さる
- 結論予告: 続きが気になる
- 既知解説: 引き込まれる
- REVEAL: 「そういうことか！」★ピーク
- 未知解説: 「ちゃんと科学的に話してる」
- まとめ表: 面白い・共有したい
- CTA: 参加欲求・遊び心

**CTA例:**
「〇〇に何を食わせたい？コメントで教えてほしい」
「こういう逸話、他に知ってたら教えてほしい」

**まとめ表例:**

| 要素 | 表の逸話 | 裏のポイント | 科学的解釈 |
|---|---|---|---|
| ... | ... | ... | ... |

## debunk（誤認訂正系）`プロテインは腎臓を悪くする等`

**コンセプト転換:** 「古い常識 vs 最新の事実」ではなく「古い常識を避けても意味ない、本当に注意すべき『？？？』を知らないと損し続ける」に転換。

**10枚構成:**

```
スライド 1（挑発）       ← `〇〇は体に悪い…それ、10年前の情弱知識！ 鍵は「？？？」`
スライド 2（自分事化）   ← `まだ〇〇を避けてませんか？…それ、損してます`
スライド 3（結論予告）   ← `結論！ 怖いのは〇〇じゃない、「？？？」だ！ 答えは後ほど→`
スライド 4〜5（既知解説）← 誤認の根拠（弱い論文）を証拠で叩く
スライド 6（REVEAL）★  ← `本当に注意すべき「？？？」の正体＝〇〇`
スライド 7〜8（未知解説）← 正しい実践の作用機序
スライド 9（まとめ表）   ← 誤認 vs 事実 vs 本当の注意点（3列）
スライド 10（CTA）
```

**口調:** このカテゴリだけキレた口調OK。

**感情設計:**
- 挑発: ドキッ・「自分のこと？」
- 自分事化: 刺さる
- 結論予告: 続きが気になる
- 既知解説: 怒りの種・「え、マジで？」
- REVEAL: 「そういうことか！」★ピーク
- 未知解説: 「助かる」・腑に落ちる
- まとめ表: 信頼・整理された
- CTA: 参加欲求・共感

**CTA例:**
「これ信じてた期間、コメントで教えてほしい（例: 3年）恥ずかしくないよ」
「周りにまだ信じてる人いる？何人くらいか教えてほしい」

**まとめ表例:**

| 誤認 | 事実 | 根拠強度 |
|---|---|---|
| ... | ... | ... |

## discovery（発見解剖系）`大谷の卵16個 / 江戸人の白米5合 等`

**構成順（10枚固定）:** ツカミ → 異常ポイント → 科学的解剖①② → 比較 → ツッコミ爆発①② → 学び → まとめ表 → CTA

「異常な事例」を見つけて、それを科学で分解する型。カテゴリ4つの中で一番「おもしろい発見をした」トーンが強い。

```
スライド 1 （ツカミ）        ← 13〜25文字。異常事実を一撃で出す
スライド 2 （異常ポイント）   ← 30〜50文字。数字でおかしさを立てる
スライド 3 （科学的解剖①）  ← 30〜50文字。1つ目のメカニズム
スライド 4 （科学的解剖②）  ← 30〜50文字。2つ目のメカニズム
スライド 5 （比較）          ← 30〜50文字。現代・一般との差を笑う
スライド 6 （ツッコミ爆発①）← 30〜50文字。セルフツッコミ必須
スライド 7 （ツッコミ爆発②）← 30〜50文字。セルフツッコミ必須
スライド 8 （学び）          ← 30〜50文字。読者への実践落とし込み
スライド 9 （まとめ表）       ← 対比表（現代 vs 対象）
スライド 10（CTA）            ← A × B = ? 数式テンプレ
```

**口調:** 他カテゴリより柔らかく。驚き・呆れ・面白がりのトーン。「ちょっと待ってくれ」「おもしろすぎるだろ！」系のセルフツッコミを6・7枚目で爆発させる。

**感情設計:**
- ツカミ: えっ？
- 異常ポイント: 好奇心
- 科学的解剖①: へぇ〜！
- 科学的解剖②: マジか！
- 比較: 差に笑う
- ツッコミ爆発①: 笑い＋驚き
- ツッコミ爆発②: 知的興奮
- 学び: 助かる
- まとめ表: 信頼
- CTA: 参加欲求

**文字数ルール（このカテゴリのみ固定）:**
- 1枚目: 13〜25文字
- 2〜8枚目: 30〜50文字
- 9枚目（まとめ表）・10枚目（CTA）: 通常ルール

**ツッコミ爆発スライドの書き方（6・7枚目）:**

事実を淡々と書いてから、セルフツッコミで落とす。SKILL.md「感情誘発の5型」のセルフツッコミ型フレーズを使う。

例:
- 「管理栄養士が電卓で弾き出した結果らしい。いや、これ考えた奴頭いいだろ！」
- 「米国は2015年に上限撤廃済。10年前の常識で恐れてる俺ら、周回遅れすぎる！」

**CTA例（discoveryは型1・型2どちらも採用可。テーマで選ぶ）:**

型1. 数式・承認欲求型（量・頻度・重量テーマ向け）:
`体重(kg) × 1日の卵個数 = ?` コメントで教えて
`週のトレ回数 × 1食の米(合) = ?` 晒してほしい

型2. 共感・仲間集め型（人物・推し・選択を晒すテーマ向け）:
`好きな格闘技の選手コメントしてくれ！！` / `俺は萩原恭平`
`推しのビルダーコメントしてくれ！！` / `俺はロニーコールマン`

**まとめ表例:**

| 要素 | 対象の戦略 | 現代科学での評価 |
|---|---|---|
| ... | ... | ... |

**適用テーマ例:**
- 固有名詞×異常な食事量（大谷の卵16個、力士のちゃんこ5000kcal）
- 歴史的事例×科学（江戸人の白米5合、飛脚の米10合）
- 動物×人間比較（マウンテンゴリラの菜食筋肥大、熊の冬眠代謝）
